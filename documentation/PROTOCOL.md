# PRE-MISIÓN Protocol: SLAMO ↔ Episodic Improver

**Version:** 1.0  
**Date:** April 2026  
**Status:** Active PRE-MISIÓN implementation  

---

## Overview

The **PRE-MISIÓN Protocol** enables SLAMO and episodic_improver to exchange mission specifications and parameter predictions via file-based JSON documents in a shared directory.

**Key Principle:** Stateless file-based communication. No ICE, no network calls. When SLAMO initiates a mission, episodic_improver can generate optimized parameters *before* execution using historical episodic learning.

---

## Directory Structure

```
episodic_memory/
├── mission_initial_{mission_id}.json    ← SLAMO writes (PRE-MISIÓN)
├── predictions_{mission_id}.json        ← episodic_improver writes
├── mission_outcome_{mission_id}.json    ← SLAMO writes (POST-MISIÓN)
├── ep_{timestamp}_{random}.json         ← Final episode for storage
└── fingerprints_index_unified_7d.json   ← 7D index (read-only)
```

## Phase 1: Pre-Mission (SLAMO → episodic_improver)

### File: `mission_initial_{mission_id}.json`

**Written by:** SLAMO  
**Read by:** episodic_improver + DirectoryMonitor  
**Trigger:** User clicks destination on map → gotoPoint() initiated  
**Latency:** Immediately (< 1ms)  

**Schema:**

```json
{
  "mission_id": "mission_1234567890123",
  "timestamp_ms": 1775753106499,
  "start_x": 10.0,
  "start_y": 15.0,
  "end_x": 50.0,
  "end_y": 45.0,
  "estimated_distance": 55.0,
  "obstacle_density": 0.45
}
```

**Field Descriptions:**

| Field | Type | Range | Description |
|-------|------|-------|-------------|
| `mission_id` | string | - | Unique mission identifier (e.g., `ep_{timestamp}_{random}`) |
| `timestamp_ms` | int | - | Unix timestamp in milliseconds |
| `start_x` | float | [-40, 40] | Robot X position in room frame (meters) |
| `start_y` | float | [-40, 40] | Robot Y position in room frame (meters) |
| `end_x` | float | [-40, 40] | Target X position (meters) |
| `end_y` | float | [-40, 40] | Target Y position (meters) |
| `estimated_distance` | float | [0, 70] | Path length estimate from planner (meters) |
| `obstacle_density` | float | [0, 1] | Obstacle density at start position (normalized) |

**Validation Rules:**

- `mission_id` must be unique across the session
- All coordinate values must be within [-40, 40] range
- `estimated_distance` should be ≥ straight-line distance
- `obstacle_density` ∈ [0, 1]

**Example (Python):**

```python
from pathlib import Path
import json

mission = {
    "mission_id": "mission_1775753106499",
    "timestamp_ms": 1775753106499,
    "start_x": 10.0,
    "start_y": 15.0,
    "end_x": 50.0,
    "end_y": 45.0,
    "estimated_distance": 55.0,
    "obstacle_density": 0.45,
}

mission_file = Path("episodic_memory") / f"mission_initial_{mission['mission_id']}.json"
mission_file.write_text(json.dumps(mission, indent=2))
```

## Phase 2: Prediction (episodic_improver → SLAMO)

### File: `predictions_{mission_id}.json`

**Generated by:** episodic_improver  
**Read by:** SLAMO  
**Trigger:** DirectoryMonitor detects `mission_initial_{mission_id}.json`  
**Latency:** ~100-500ms (depends on index size)  

**Schema:**

```json
{
  "mission_id": "mission_1234567890123",
  "timestamp_ms": 1775753106599,
  "status": "ready",
  "fingerprint_7d": [0.125, 0.1875, 0.333, 0.618, 1.05, 0.45, 0.473],
  "best_match_id": "ep_1773938016714",
  "best_match_similarity": 0.87,
  "predicted_parameters": {
    "base_speed": 0.35,
    "max_adv_speed": 0.75,
    "angular_velocity": 0.95,
    "angular_acceleration": 1.5,
    "accel_limit": 0.25,
    "decel_limit": 0.30
  },
  "search_results": [
    {
      "rank": 1,
      "episode_id": "ep_1773938016714",
      "similarity_score": 0.87
    },
    {
      "rank": 2,
      "episode_id": "ep_1773937012345",
      "similarity_score": 0.82
    },
    {
      "rank": 3,
      "episode_id": "ep_1773936789012",
      "similarity_score": 0.78
    }
  ],
  "perturbation": {
    "parameter": "angular_velocity",
    "sigma": 0.0475,
    "strategy": "adaptive_explore"
  }
}
```

**Field Descriptions:**

| Field | Type/Schema | Description |
|-------|------------|-------------|
| `mission_id` | string | Echo back mission_id for correlation |
| `timestamp_ms` | int | When predictions were generated |
| `status` | enum | `"ready"` \| `"error"` \| `"timeout"` |
| `fingerprint_7d` | float[7] | 7D fingerprint computed from mission_initial |
| `best_match_id` | string | Episode ID of most similar historical mission |
| `best_match_similarity` | float | Cosine similarity ∈ [0, 1] (1 = identical) |
| `predicted_parameters` | dict | Control parameters to use (key = param name, value = float) |
| `search_results` | object[3] | Top-3 similar episodes (metadata only) |
| `perturbation` | object | Info about parameter perturbation applied |

**Fingerprint 7D Interpretation:**

```
[f1, f2, f3, f4, f5, f6, f7]
 ↓   ↓   ↓   ↓   ↓   ↓   ↓
[pos_x, pos_y, heading, distance, tortuosity, density, complexity]
```

See [DOCUMENTACION_7D.md](DOCUMENTACION_7D.md) for full 7D descriptor definitions.

**Status Codes:**

- `"ready"`: Predictions generated successfully, use `predicted_parameters`
- `"error"`: Error generating predictions, ignore `predicted_parameters`
- `"timeout"`: No similar episodes found, use fallback parameters

**Example (SLAMO C++):**

```cpp
// (1) Read predictions
std::ifstream predictions_file("episodic_memory/predictions_" + mission_id + ".json");
json predictions = json::parse(predictions_file);

// (2) Check status
if (predictions["status"] == "ready") {
    // (3) Extract predicted parameters
    auto params = predictions["predicted_parameters"];
    
    // (4) Apply to trajectory controller
    trajectory_controller_.set_max_velocity(params["max_adv_speed"]);
    trajectory_controller_.set_angular_velocity(params["angular_velocity"]);
    
    logger.info(f"Applied predicted params (similarity={predictions['best_match_similarity']})");
}
```

## Phase 3: Mission Execution (SLAMO)

**What happens:**
- SLAMO reads `predictions_{mission_id}.json` (if exists)
- Applies predicted parameters to trajectory controller (or uses defaults if predictions failed)
- Executes navigation with optimized parameters
- Monitors mission state (time, collisions, blocked events, etc.)

## Phase 4: Post-Mission (SLAMO → episodic_improver)

### File: `mission_outcome_{mission_id}.json`

**Written by:** SLAMO  
**Read by:** episodic_improver (future episode processing)  
**Trigger:** Mission completion (success, timeout, failure)  
**Latency:** Immediately after mission ends  

**Schema:**

```json
{
  "mission_id": "mission_1234567890123",
  "timestamp_ms": 1775753148599,
  "status": "success",
  "time_to_goal_s": 42.5,
  "collisions": 0,
  "blocked_time_s": 0.0,
  "composite_score": 87.3,
  "predicted_from_episode": "ep_1773938016714",
  "similarity_score": 0.87,
  "control_params": {
    "base_speed": 0.35,
    "max_adv_speed": 0.75,
    "angular_velocity": 0.95,
    "angular_acceleration": 1.5,
    "accel_limit": 0.25,
    "decel_limit": 0.30
  }
}
```

**Field Descriptions:**

| Field | Type | Description |
|-------|------|-------------|
| `mission_id` | string | (Required) Unique mission identifier |
| `timestamp_ms` | int | When mission completed |
| `status` | enum | `"success"` \| `"failure"` \| `"timeout"` |
| `time_to_goal_s` | float | Duration in seconds |
| `collisions` | int | Number of collision events |
| `blocked_time_s` | float | Time blocked/stuck in seconds |
| `composite_score` | float | Overall quality score [0, 100] |
| `predicted_from_episode` | string | *(Optional)* Episode ID predictions came from |
| `similarity_score` | float | *(Optional)* Similarity used for prediction |
| `control_params` | dict | *(Optional)* Parameters actually used |

## Complete Flow Diagram

```
┌──────────────────────────┐
│ USER CLICKS ON MAP       │
│ (gotoPoint() invoked)    │
└────────────┬─────────────┘
             ↓
      ┌─────────────────────────────────────────────┐
      │ [PHASE 1] SLAMO Pre-Mission                │
      │ • Compute path & estimated_distance        │
      │ • Measure obstacle_density at start         │
      │ • Write mission_initial_{mid}.json          │
      └────────────┬────────────────────────────────┘
                   ↓
    ┌──────────────────────────────────────┐
    │ episodic_improver Watchdog detects   │
    │ mission_initial_.json                │
    └────────────┬─────────────────────────┘
                 ↓
     ┌───────────────────────────────────────────┐
     │ [PHASE 2] episodic_improver Prediction   │
     │ • Extract 7D fingerprint                 │
     │ • Search k-NN (top-3 similar)            │
     │ • Select best-match                      │
     │ • Perturb parameters adaptively          │
     │ • Write predictions_{mid}.json           │
     └────────────┬────────────────────────────┘
                  ↓
        ┌──────────────────────────┐
        │ [PHASE 3] SLAMO           │
        │ • Read predictions file   │
        │ • Apply predicted params  │
        │ • Navigate with optimized │
        │   parameters              │
        └────────────┬─────────────┘
                     ↓
        ┌──────────────────────────┐
        │ • Monitor execution      │
        │ • Collect metrics        │
        │ • Record outcome         │
        └────────────┬─────────────┘
                     ↓
      ┌──────────────────────────────┐
      │ [PHASE 4] SLAMO Post-Mission │
      │ • Write mission_outcome_     │
      │   {mid}.json                 │
      │ • Merge with mission_initial │
      │ • Save as episode JSON       │
      └──────────────────────────────┘
                     ↓
    ┌──────────────────────────────────────┐
    │ episodic_improver processes episode  │
    │ • Detects ep_*.json                  │
    │ • Computes 7D fingerprint (geometric + normalized) │
    │ • Updates index for next mission     │
    └──────────────────────────────────────┘
```

## Timing & Guarantees

### Pre-Mission Latency

| Stage | Time | Notes |
|-------|------|-------|
| SLAMO writes mission_initial | ~1ms | File I/O |
| Watchdog detects file | ~10-50ms | depends on poll interval |
| 7D prediction computation | ~50-200ms | ~189 episodes in index |
| Write predictions file | ~1ms | File I/O |
| **Total** | **~100-300ms** | Conservative estimate |

**SLAMO should timeout if no predictions after ~5 seconds** and fall back to default parameters.

### Post-Mission Latency

No latency guarantees. episodic_improver processes episodes asynchronously.

## Error Handling

### SLAMO Perspective

```python
predictions = bridge.get_predictions(...)

if predictions is None:
    # Timeout - use default parameters
    logger.warning("Timeout waiting for predictions, using defaults")
    use_default_parameters()
    
elif predictions.status != "ready":
    # Error generating predictions
    logger.warning(f"Prediction failed: {predictions.status}")
    use_default_parameters()
    
else:
    # Success - apply predicted parameters
    logger.info(f"Applying predictions from {predictions.best_match_id}")
    apply_parameters(predictions.predicted_parameters)
```

### episodic_improver Perspective

If unable to generate predictions, leave `predictions_{mission_id}.json` unwritten and log error. SLAMO will timeout naturally.

## Appendix: Field Ranges

**Coordinates (Room Frame):**
- X: [-40, 40] meters (room width)
- Y: [-40, 40] meters (room depth)

**Distances:**
- estimated_distance: [0, 70] meters (max diagonal ≈ 113m but typical missions are shorter)

**Normalized Metrics:**
- obstacle_density: [0, 1] (0=empty, 1=fully blocked)
- similarity_score: [0, 1] (0=completely different, 1=identical)

**Control Parameters:**
- base_speed: [0.1, 2.0] m/s
- max_adv_speed: [0.1, 2.0] m/s
- angular_velocity: [0.2, 3.0] rad/s
- angular_acceleration: [0.1, 1.0] rad/s²
- accel_limit: [0.1, 1.0] m/s²
- decel_limit: [0.1, 1.0] m/s²
